3Introducción
4
5En Swift, los arrays y diccionarios son herramientas fundamentales para organizar datos.
6Mientras que los diccionarios ofrecen ergonomía para acceder y modificar elementos por clave, 
7los arrays son el tipo más común en aplicaciones que recuperan datos de apis remotas o
8soluciones de almacenamiento locales.
9
10En este artículo exploraremos cómo mejorar la manipulación de arrays utilizando subscripts
11personalizados para obtener la simplicidad de un diccionario.
12
13Manipulación de diccionarios
14
15En Swift, manipular los datos de un diccionario es bastante fácil,
16por ejemplo, si tuvieramos un modelo como este:
17
18struct Model {
19    let id: Int 
20    var name: String?
21}
22
23Y un diccionario cuyas claves fuesen del mismo tipo  que el `id`
24de ese modelo:
25
26var dict = [Int: Model]()
27
28Podríamos realizar acciones CRUD de una manera muy sencilla:
29
30C: dict[1] = Model(id: 1, name: "Some name")
31R: _ = dict[1] 
32U: dict[1]?.name = "New name"
33D: dict[1] = nil
34
35Manipulación de arrays
36
37Con un array, manipularíamos los datos de una manera más verbosa:
38
39var array = [Model]()
40
41C: array.append(Model(id: 1, name: "Some name"))
42
43(podemos leer un elemento en el array de varias maneras)
44
45R: _ = array.filter { $0.id == 1 }.first
46R: _ = array[0] (inseguro, puede crashear)
47R: 
48if let idx = array.firstIndex(where: {$0.id == 1}) {
49    _ = array[idx]
50}
51
52U: array[0].name = "Some name" (inseguro)
53U:
54if let idx = array.firstIndex(where: {$0.id == 1}) {
55    array[idx].name = "Some name"
56}
57
58D:
59if let idx = array.firstIndex(where: {$0.id == 1}) {
60    array.remove(at: idx)
61}
62
63Usando un subscript personalizado:
64
65Cómo vemos, la ergonomía de un diccionario es mucho más práctica
66y concisa, especialmente en los casos de actualización y eliminación,
67en los que que con un array necesitamos un paso previo de recuperación
68del índice del elemento a actualizar/eliminar
69
70Swift nos permite crear métodos `subscript` personalizados para 
71manipular colecciones.
72
73Gracias a esta funcionalidad, podemos obtener una api de manipulación 
74de arrays idéntica a la de un diccionario.
75
76El único requisito es que los elementos de nuestro array conformen al
77protocolo `Identifiable`.
78
79Implementación:
80
81El subscript encapsula el boilerplate de creación, lectura, actualización
82y eliminación que generalmente declaramos de forma manual con un array:
83
84extension Array where Element: Identifiable {
85    
86    Toma come clave el id de sus elementos...
87    
88    subscript(id: Element.ID) -> Element? {
89        
90        ... y al igual que un diccionario, devuelve un opcional...
91        
92        
93        Lectura:
94        
95        get { first { $0.id == id } }
96        set(newValue) {
97            
98            Actualización/Eliminación:
99            Si ya existe un elemento con el mismo ID, actualizamos
100            o eliminamos en función de si el nuevo valor es nulo o no:
101            
102            if let index = firstIndex(where: { $0.id == id }) {
103                if let newValue = newValue {
104                    self[index] = newValue
105                } else {
106                    remove(at: index)
107                }
108                
109                Creación
110                Si no existe, simplemente lo añadimos al array:
111                
112            } else if let newValue = newValue {
113                append(newValue)
114            }
115        }
116    }
117}
118
119Uso:
120
121struct Todo: Identifiable {
122    let id = UUID()
123    var description: String
124    var isChecked = false
125    
126    init(_ description: String) {
127        self.description = description
128    }
129}
130
131final class TodoStore: ObservableObject {
132    
133    @Published private(set) var data = [Todo]()
134    
135    😎 Con subscript:
136    
137    func check_1(_ id: UUID) {
138        data[id]?.isChecked.toggle()
139    }
140    
141    😫 Sin subscript:
142    
143    func check_2(_ id: UUID) {
144        if let index = data.firstIndex(where: { $0.id == id }) {
145            data[index].isChecked.toggle()
146        }
147    }
148    
149    🎉 Más ejemplos:
150    
151    func upsert(item: Todo) {
152        data[item.id] = item
153    }
154    
155    func read(id: UUID) -> Todo? {
156        data[id]
157    }
158    
159    func delete(id: UUID) {
160        data[id] = nil
161    }
162}

213Conclusión
214
215Este pequeño snippet permite obtener varias ventajas, entre ellas:
216•	Reducción del código repetitivo: Elimina la necesidad de buscar índices manualmente para operaciones comunes.
217•	Ergonomía: Ofrece una API similar a un diccionario.
218•	Flexibilidad: Combina lo mejor de ambas estructuras de datos.
